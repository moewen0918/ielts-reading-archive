# 📖 阅读精读特训档案室 · IELTS AI Coach

一个纯前端、单文件运行的雅思阅读精读复盘工具。没有后端、没有数据库，所有数据都存在你自己浏览器的 `localStorage` 里，打开即用，部署到 GitHub Pages 就能在任何设备上访问。

你可以把它理解成：一本会自动帮你批改、统计、出题的"错题本"，专门为雅思阅读精读设计。

---

## ✨ 这个工具能做什么

### 🧠 AI 智能精读解构
把你精读时圈出的生词、短语、长难句打包丢给 AI（支持 Google Gemini 或 DeepSeek 双引擎切换），AI 会按固定格式拆解输出：

- **词汇档案**：音标、词性、语境精准释义，外加雅思真题最常考的 3-4 个同义替换词对照表
- **长难句解剖**：每句话独立列出——先给英文原句，再给权威翻译、语法骨架拆解、核心考点词标注、雅思防坑指南，最后附一个"仿写练习"帮你活学活用句型

### 🎯 弱点雷达（真实错误率）
精读时可以给错题打上题型标签（判断题 TFN / 配对题 / 填空题 / 单选题 / 标题配对 / 其他），并登记这次"错几题 / 共几题"。系统会汇总你所有历史记录，算出**每个题型的真实错误率**（不是简单的次数占比），一眼看出哪类题目在拖后腿。

### 📚 体裁表现对比
文章可以打上体裁标签（人文 / 社科 / 自然科学 / 科技），仪表盘会对比你在不同体裁上的精读篇数、篇均生词量、错误率——雅思阅读的背景知识差异很大，这能帮你看清哪类话题的知识储备还不够。

### 🃏 生词闪卡库（间隔重复记忆）
AI 解析出的生词会自动整理成可翻转的闪卡，正面单词，背面释义 + 出处例句 + 同义替换词。复习时用简化版 SM-2 间隔重复算法（忘记了 / 模糊 / 记得 / 很简单 四档反馈），系统自动安排下次复习时间，默认只展示"今日待复习"的卡片，也可以随时切到"全部卡片"浏览。也支持手动快速加词，不用非得走完整分析流程。

### 🔁 二刷对比
同一篇文章精读第二次、第三次时，系统会自动识别，弹出对比提示：错误率涨了还是跌了、用时是变快还是变慢，让"进步"有据可查，而不只是感觉。

### 🔥 学习节奏
仪表盘会根据你真实的操作时间戳，算出连续打卡天数，以及近 7 天每天的学习次数柱状图——督促自己保持节奏，而不是躺在那当摆设的假数据。

### ⏱ 计时 + 🔊 朗读
分析前可以开始计时，自动记录这次复盘用了多久；AI 拆解出的每个英文原句旁边都有朗读按钮，用浏览器自带语音合成读出来，顺便练发音。

### 📑 一键生成周报
把最近 7 天的复盘记录打包，交给 AI 扮演"雅思首席考官"写一段深度复盘评语（表现总览 / 思维误区诊断 / 下周训练建议），支持下载成 Markdown。

### 💾 备份与恢复
一键导出所有复盘记录和生词卡为 `.json` 文件，换设备后一键导入合并，数据不丢失。出于安全考虑，备份文件不包含你的 AI API Key，换设备需要重新填写一次。

### 🔖 浏览器随手记（配合 Tampermonkey）
配套提供了一个油猴脚本：在任意网页（Economist、BBC、Wikipedia 等）阅读时选中生词或句子，累积打包后一键跳转回这个档案室，自动填好文章标题、Part 章节和正文内容，不用手动复制粘贴。

## 🚀 快速开始

1. 打开网页后，先在顶部选择 AI 引擎（Gemini 或 DeepSeek），填入你自己的 API Key
   > Key 只存在你自己浏览器本地，不会上传到任何服务器
2. 把要精读的语料（生词 + 长难句，一行一个）交给工作区，或者用配套油猴脚本从外部网页自动导入
3. 选好这篇文章的 Part、体裁，点"开始计时"，读完后点"唤醒 AI 启动智能多维解构"
4. 提交前记得给错题打上题型标签、填上"错几题/共几题"，弱点雷达才会准确
5. 定期去"生词闪卡库"复习到期的卡片，每周点一次"生成本周特训报告"看整体表现

## 🗂 数据存储说明

所有数据都保存在浏览器 `localStorage` 中，不依赖任何后端服务器：

| Key | 内容 |
|---|---|
| `ielts_db_pro_v5` | 每次精读的完整记录（原文、AI 解析结果、错题统计、体裁、用时等） |
| `ielts_flashcards_v1` | 生词闪卡库（含间隔重复的复习状态） |
| `active_engine` / `gemini_api_key` / `deepseek_api_key` 等 | AI 引擎与 Key 的本地配置 |

清空浏览器缓存、更换设备或换浏览器都会导致数据丢失，**请定期使用"导出备份"功能保存数据**。

## ⚠️ 使用须知

- 本工具是纯前端页面，AI 请求直接从你的浏览器发往 Google / DeepSeek 官方接口，不经过任何中间服务器，也因此需要你自己承担 API 调用产生的费用
- 部分地区访问 Gemini / DeepSeek 接口可能需要网络代理
- 请勿在公开场合分享你导出的备份文件中如果不小心包含了敏感个人信息的部分（正常情况下备份不含 API Key）

纯 HTML + CSS + 原生 JavaScript 单文件实现，无需构建、无需安装依赖，直接部署到 GitHub Pages 即可访问。字体使用 Source Serif 4 / Inter / IBM Plex Mono，图标库使用 Phosphor Icons。
